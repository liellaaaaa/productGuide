# 美食/超市导购助手 — 需求记录

## 一、项目目标

用 Python 搭建一个**结构化的美食/超市商品导购**小程序，验证 **RAG（Chroma 向量库）+ 大模型生成** 的可行流程。

---

## 二、需求梳理

| 编号 | 需求 | 说明 |
|------|------|------|
| 1 | 交互方式 | 仅在**命令行**输出结果，**不需要** Web/桌面前端。 |
| 2 | 知识库 | 使用 **Chroma** 作为向量存储与检索后端。 |
| 3 | 大模型 | 接入**豆包**，通过火山引擎 **Ark** SDK（`volcenginesdkarkruntime`）调用；模型示例：`doubao-seed-2-0-lite-260215`；API Key 从环境变量 **`ARK_API_KEY`** 读取；支持**流式**输出（按 chunk 打印 `delta.content`）。 |
| 4 | 日志 | **用户输入与模型输出**以 **JSON** 形式**持久化保存**（日志），便于复现与调试。 |

---

## 三、需求分析（实现要点）

1. **边界**：单进程 CLI 工具即可；核心是「查库 → 拼 prompt → 调 Ark 流式生成 → 终端展示」。
2. **RAG**：文档入库（embedding）+ 查询时 top-k 检索，将检索片段注入 system/user 提示词，再交给豆包生成导购建议。
3. **豆包接入**：依赖 `Ark` 客户端、`ARK_API_KEY`；流式循环处理空 `choices` 与 `delta.content` 为空的情况，结束时换行。
4. **日志**：每次交互写一条或一组 JSON 记录（含时间、用户问题、检索摘要可选、模型完整回复拼接结果等），落盘到指定目录或单一日志文件。

---

## 四、后续实现顺序（建议）

1. 依赖与环境变量说明（`requirements.txt`、`ARK_API_KEY`）。  
2. Chroma 建库与示例商品/美食结构化数据入库。  
3. 检索 + Ark 调用与 CLI 主循环。  
4. JSON 日志写入。
