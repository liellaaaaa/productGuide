## 3. 运行时调用链（细化）

1. **入口** `python -m product_guide`（或约定包名）解析 CLI，加载 `config`。  
2. **`chat` 子命令**：读用户输入 → `kb.query(question, top_k)` → `prompt.build(system_tpl, contexts, question)` → `ark_client.stream_chat(messages)` → 逐 chunk 写 **stdout**，拼接完整文本。  
3. **`ingest` 子命令**：读 `data/` 下结构化源 → `ingest.chunk(...)` → `kb.embed_and_upsert(...)`。  
4. **每次 `chat` 一轮结束后**：`json_log.append(record)` 写一条 JSON（或 JSON Lines 一行）。

依赖方向保持单向：`cli` → `config, kb, prompt, ark_client, json_log`；`ingest` → `kb`；**禁止** `kb` 依赖 `cli`。

## 4. 包结构与模块接口（建议）

以下名称与 `02概要设计.md` §8 对齐，实现时可微调文件名，但职责不变。

### 4.1 `config.py`

- **职责**：集中读取环境变量与可选默认值；对外暴露简单对象或命名空间（如 `Config` dataclass）。  
- **必选**：`ark_api_key: str`，来自 `os.environ["ARK_API_KEY"]`，缺失时**启动即报错退出**（stderr 提示，不打印密钥）。  
- **可选**（均有文档化默认值）：  

| 变量名 | 含义 | 示例默认值 |
|--------|------|--------------|
| `CHROMA_PATH` | Chroma 持久化目录 | `./chroma_db` |
| `LOG_PATH` | JSON 日志文件路径，或目录（见 §9） | `./logs/session.jsonl` |
| `ARK_MODEL` | 豆包模型 ID | `doubao-seed-2-0-lite-260215` |
| `TOP_K` | 检索条数 | `5` |
| `EMBEDDING_MODEL` 或等价 | 与 Chroma 一致的 embedding 标识 | 实现时与 `kb`/`ingest` 共用同一常量 |

### 4.2 `kb.py`（Chroma 封装）

- **职责**：`get_or_create_collection`；查询时与用户问题使用**同一 embedding 管线**；返回 `list[str]` 或 `list[dict]`（含 `document`/`metadata` 若需调试）。  
- **接口建议**：  
  - `ensure_collection()`  
  - `query(text: str, n_results: int) -> list[str]`  
  - `upsert(ids, documents, metadatas=None)`（供 ingest）  
- **持久化**：`PersistentClient` + `path=config.chroma_path`。

### 4.3 `ingest.py`

- **职责**：将 `data/` 下结构化条目（JSON/CSV 等，实现时固定一种）解析为文档块；调用 `kb` 写入；**幂等策略**约定为：同一 `id` 覆盖写入或先删后插（实现选一种并文档说明）。  
- **分块规则**：条目级一条文档即可（验证阶段可粗粒度）；字段可拼接为可读文本供 embedding。

### 4.4 `prompt.py`

- **职责**：仅字符串组装，无 I/O。  
- **输出**：`build_messages(retrieved_chunks: list[str], user_question: str) -> list[dict]`，兼容 Ark `messages` 格式（`role` + `content`）。  
- **system 内容要点**：固定「美食/超市导购」角色；说明检索片段仅供参考；**无检索结果**时由调用方传入空列表，`prompt` 在 system 或 user 中明确写「当前知识库未检索到直接相关内容」（与需求「仍可调用模型」一致）。

### 4.5 `ark_client.py`

- **职责**：`Ark(api_key=config.ark_api_key)`；封装**流式** `chat.completions.create(..., stream=True)`（具体 API 以 SDK 为准）。  
- **生成器接口建议**：`stream_chat(messages) -> Iterator[str]`，内部对每个 chunk：  
  - 若 `choices` 为空 → 跳过；  
  - 若 `delta.content` 为空或缺失 → 跳过；  
  - 否则 yield 文本片段。  
- **调用方**（`cli`）负责：`print(chunk, end="", flush=True)`，迭代结束后 `print()` 换行。

### 4.6 `json_log.py`

- **职责**：将单条记录序列化为 JSON 并**追加**写入；推荐 **JSON Lines**（每行一个 JSON 对象），便于流式追加与 `tail`。  
- **线程/进程**：单进程 CLI 下普通追加即可；若多会话并发需文档声明「本阶段单进程」。

### 4.7 `cli.py`

- **职责**：`argparse`：`ingest`、`chat`（默认可指向 `chat`）；`chat` 内 REPL 循环或单次 `-q` 问题（实现二选一或并存，README 说明）。  
- **stderr**：API/网络错误、配置错误；**stdout**：仅模型流式正文（便于管道重定向）。

## 5. 提示词结构（冻结行为）

| 消息 | 内容要点 |
|------|----------|
| system | 角色：结构化美食/超市导购助手；遵守检索片段为辅助依据；无匹配时诚实说明并仍给出通用建议。 |
| user | 可选前缀：`【知识库检索片段】\n` + 片段拼接（编号或分隔符）；其后为用户原问题。 |

片段拼接示例（逻辑，非定稿字面）：多个 chunk 用 `\n---\n` 分隔；`top_k` 来自配置。

## 6. 知识库数据约定

| 项 | 约定 |
|----|------|
| 源数据位置 | 仓库内 `data/`，格式在 README 与 ingest 中写明。 |
| `collection` 名 | 固定常量，如 `food_retail_guide`，避免与多项目混用同目录时冲突。 |
| 文档 `id` | 稳定主键（如 SKU 或行号哈希），便于重复 ingest。 |
| metadata（可选） | `source`、`category` 等，便于日志「检索摘要」展示。 |

## 7. JSON 日志字段（最小集与扩展）

每条记录**至少**包含：

| 字段 | 类型 | 说明 |
|------|------|------|
| `ts` | string | ISO8601 时间戳（UTC 或本地，文档固定一种）。 |
| `user_input` | string | 用户问题原文。 |
| `model_output` | string | 流式片段拼接后的**完整**回复。 |

**建议**包含（满足「检索摘要」与调试）：

| 字段 | 类型 | 说明 |
|------|------|------|
| `retrieval_summary` | string 或 array | 检索到的片段摘要或标题列表；无结果时显式 `"none"` 或 `[]`。 |
| `model` | string | 实际调用的模型 ID。 |

可选：`duration_ms`、`top_k`、`error`（失败时）。

## 8. 异常与边界行为

| 场景 | 行为 |
|------|------|
| `ARK_API_KEY` 未设置 | 启动失败，stderr 一行说明。 |
| Chroma 路径不存在 | 首次 `ingest` 创建；仅 `chat` 且库为空时仍可调用模型，prompt 标明无检索结果。 |
| 检索为空 | 不调异常；prompt 注明无相关内容；日志 `retrieval_summary` 反映空。 |
| Ark 流式中断/HTTP 错误 | stderr 输出错误信息；该轮可不写或写带 `error` 字段的日志（实现固定一种）。 |
| chunk 空 `choices` / 空 `delta` | 忽略，继续；若全程无内容，完整回复可为空字符串，日志照实记录。 |

密钥与响应体中的敏感信息**不得**写入日志明文（本场景默认无用户隐私字段，仅问题与回复）。

## 9. 验收与自测要点（映射需求 §7）

| 需求验收 | 详细设计检查项 |
|----------|----------------|
| CLI 流式导购 | `ark_client` yield + `cli` 打印；结束换行。 |
| Chroma 参与生成 | `kb.query` 结果进入 `prompt.build`。 |
| Ark + 环境变量 + chunk 处理 | `config` 读 `ARK_API_KEY`；空 chunk 跳过。 |
| JSON 落盘 | `json_log` 字段含 `ts`、`user_input`、`model_output`；路径可配置。 |
